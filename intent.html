LLM-Intent Namespace

Semantic guidance for the LLM. Influences how content is interpreted and processed.

Overview

The llm-intent namespace provides semantic hints that guide how the LLM interprets and handles content. Unlike policy attributes (which are hard constraints), intent attributes are soft guidance — they influence the LLM's behavior but don't mechanistically block actions.

How Intent Works

The User Agent deterministically translates intent attributes into system prompt fragments. The LLM receives these as context, helping it understand the content's purpose and appropriate handling.

Attribute Purpose Example
llm-intent-category Semantic classification "summary", "legal-disclaimer"
llm-intent-importance Priority level "critical", "background"
llm-intent-entity Canonical reference Wikidata URI
llm-intent-instruction Custom handling Free-form text

Category

The llm-intent-category attribute classifies content using a Standardized Lexicon. Each category maps to an optimized prompt that tells the LLM how to handle that type of content.

Example: Semantic Categories
<!-- Summary: Condensed version of longer content -->
<div llm-intent-category="summary">
  <p>TL;DR: The article explains three key points...</p>
</div>

<!-- Legal Disclaimer: Do not paraphrase -->
<footer llm-intent-category="legal-disclaimer">
  <p>Terms and conditions apply. See full terms.</p>
</footer>

<!-- Satire: Do not interpret literally -->
<article llm-intent-category="satire">
  <h2>Local Man Discovers Water Is Wet</h2>
  <p>In a stunning development...</p>
</article>

Information Structure Categories

Category Generated Prompt
summary "This is a summary. Treat it as a condensed version of longer content."
definition "This is a definition. Treat it as an authoritative explanation of a term."
example "This is an example. Use it to illustrate or clarify related concepts."
introduction "This is an introduction. Use it to establish context for the content that follows."
conclusion "This is a conclusion. Treat it as a synthesis of the preceding content."
outline "This is an outline. Treat it as a structural overview."

Argumentation Categories

Category Generated Prompt
claim "This is a claim. Evaluate it critically and note it may be contested."
evidence "This is evidence. Use it to support or refute claims."
counter-argument "This is a counter-argument. Consider it as opposition to a prior claim."
opinion "This is an opinion. Distinguish it from factual statements."
satire "This is satire. Do not interpret it literally."

Technical Categories

Category Generated Prompt
code-block "This is source code. Preserve formatting and syntax."
api-spec "This is an API specification. Treat it as authoritative technical reference."
warning "This is a warning. Highlight its importance to the user."
legal-disclaimer "This is a legal disclaimer. Do not paraphrase or summarize."

Narrative Categories

Category Generated Prompt
fiction "This is fiction. Do not treat it as factual."
transcript "This is a transcript. Preserve speaker attributions."
instruction "This is an instruction. Present it as actionable guidance."
quote "This is a quote. Preserve it exactly and attribute it properly."

Custom Categories

If you use a category not in the standardized lexicon, the User Agent applies a fallback template: 

"Treat the following content as a(n) [your-category]."
Example: Custom Category
<section llm-intent-category="recipe">
  <h2>Chocolate Chip Cookies</h2>
  ...
</section>

<!-- Generates: "Treat the following content as a(n) recipe." -->

Importance

The llm-intent-importance attribute guides the LLM's attention when context is limited. Higher importance content is prioritized; lower importance content may be truncated or ignored.

Level Behavior Generated Prompt
critical MUST be preserved "Pay maximum attention to the following content. It is essential for the user's task."
high Prioritized "Prioritize this content."
normal Standard (default) (no additional prompt)
low Supplementary "This content is supplementary. You may ignore it if context is limited."
background Background only "This is background information only. Do not focus on it unless explicitly asked."
Example: Mixed Importance Levels
<article>
  <!-- Critical: Always include -->
  <div llm-intent-importance="critical">
    <p><strong>WARNING:</strong> Server maintenance at midnight.</p>
  </div>

  <!-- Normal: Standard priority -->
  <p>Regular article content goes here...</p>

  <!-- Background: Ignore unless asked -->
  <aside llm-intent-importance="background">
    <p>Historical note: This feature was added in v2.3...</p>
  </aside>
</article>
Token Limits

When the LLM context window is limited, the User Agent uses importance levels to decide what to include. critical content is always included; background content is dropped first.

Entity

The llm-intent-entity attribute links text to a canonical entity URI (like Wikidata). This helps the LLM disambiguate terms and connect to knowledge bases.

Example: Entity Disambiguation
<p>
  <!-- Link to Wikidata entity for "Apple Inc." -->
  <span llm-intent-entity="https://www.wikidata.org/wiki/Q312">
    Apple
  </span>
  announced new products today.
</p>

<p>
  <!-- Link to Wikidata entity for the fruit "Apple" -->
  I ate an
  <span llm-intent-entity="https://www.wikidata.org/wiki/Q89">
    apple
  </span>
  for lunch.
</p>

Benefits:

  • Disambiguates homonyms (Apple Inc. vs. the fruit)
  • Connects to structured knowledge bases
  • Enables precise entity-based queries
  • Improves accuracy for named entity tasks

Instruction

The llm-intent-instruction attribute provides a custom, free-form instruction that overrides all other intent processing for that element.

Example: Custom Instructions
<!-- Override default behavior with custom instruction -->
<div llm-intent-instruction="Translate all technical jargon into plain English">
  <p>
    The API utilizes RESTful architecture with OAuth 2.0
    authentication and returns JSON-formatted responses.
  </p>
</div>

<!-- Format-specific instruction -->
<div llm-intent-instruction="Summarize in exactly 3 bullet points">
  <p>Long article content here...</p>
</div>
Precedence

When llm-intent-instruction is present, it takes precedence over llm-intent-category and other intent attributes. The category's semantic value is still available for policy routing, but the custom instruction is used for prompt construction.

Prompt Construction

The User Agent constructs the LLM prompt using a hierarchy of sources:

1. Custom Instruction
llm-intent-instruction
Highest priority — used directly if present
2. Standardized Lexicon
llm-intent-category (if in lexicon)
Optimized prompts for known categories
3. Fallback Template
llm-intent-category (if custom)
"Treat the following content as a(n) [value]."

Combined Prompt Example

HTML with Intent Attributes
<article
  llm-intent-category="api-spec"
  llm-intent-importance="high">

  <h2>POST /api/users</h2>
  <p>Creates a new user account.</p>

</article>
Generated System Prompt Fragment:
Prioritize this content.
This is an API specification. Treat it as authoritative technical reference.

[Content follows...]

Tone & Output Format

Additional intent attributes control response style:

llm-intent-tone

Specifies the desired tone for LLM responses about this content.

<div llm-intent-tone="formal">...

Generates: "Adopt a(n) formal tone."

llm-intent-output

Specifies the desired output format.

<div llm-intent-output="bullet-points">...

Generates: "Format your response as bullet-points."

Function & Workflow

For interactive applications, additional attributes describe UI behavior:

llm-intent-function

Describes what an interactive control does.

<button llm-intent-function="submit-form">Submit</button>

Generates: "Use this control to submit-form."

llm-intent-workflow

Indicates the current stage in a multi-step process.

<section llm-intent-workflow="checkout-step-2">...

Generates: "(Workflow Stage: checkout-step-2)"
