Examples

This page provides complete, real-world examples of LLM Markup in action. Each example demonstrates common patterns and best practices.

News Article

A typical news article with protected quotes, annotatable body text, and hidden advertising/tracking elements.

news-article.html
<article
  llm-provenance-source="https://news.example/article/12345"
  llm-provenance-citation="TechNews Daily">

  <!-- Header: Important, read-only -->
  <header llm-intent-importance="high">
    <h1>Major Tech Company Announces New Product</h1>
    <p class="byline">By Jane Smith • January 15, 2024</p>
  </header>

  <!-- Summary: Critical for context -->
  <div llm-intent-category="summary"
       llm-intent-importance="critical">
    <p>The company unveiled its latest product at the annual
    conference, featuring improved performance and new AI capabilities.</p>
  </div>

  <!-- Body: Can be annotated (highlights, links) but not edited -->
  <div class="article-body" llm-policy-output="annotation style">
    <p>
      In a keynote presentation that lasted over two hours, CEO
      <span llm-intent-entity="https://wikidata.org/wiki/Q...">
        John Doe
      </span>
      demonstrated the new features to an audience of thousands.
    </p>

    <!-- Quote: Strictly read-only -->
    <blockquote
      llm-policy-output="readonly"
      llm-intent-category="quote"
      llm-provenance-citation="John Doe, CEO">
      <p>"This is the most significant product we've ever built.
      It will change how people work."</p>
    </blockquote>

    <p>Industry analysts were cautiously optimistic...</p>
  </div>

  <!-- Comments: Append-only (can add, not edit existing) -->
  <section class="comments" llm-policy-output="append">
    <h2>Comments</h2>
    <div class="comment" llm-policy-output="readonly">
      <p>Existing comment that cannot be modified.</p>
    </div>
    <!-- LLM can add new comments here -->
  </section>
</article>

<!-- Advertisement: Hidden from LLM -->
<aside class="ad" llm-policy-input="none">
  <img src="ad.jpg" alt="Advertisement">
</aside>

E-Commerce Product Page

A product page with editable descriptions, protected pricing, and form auto-fill capabilities.

product-page.html
<main itemscope itemtype="https://schema.org/Product">

  <!-- Product name: Editable for personalization -->
  <h1 itemprop="name" llm-policy-output="content">
    Widget Pro Max
  </h1>

  <!-- Price: NEVER editable -->
  <div class="pricing" llm-policy-output="readonly">
    <span itemprop="price">$99.99</span>
    <meta itemprop="priceCurrency" content="USD">
  </div>

  <!-- Description: Summarizable and annotatable -->
  <div itemprop="description"
       llm-policy-output="annotation"
       llm-intent-category="definition">
    <p>
      The Widget Pro Max features our most advanced technology yet.
      With 50% more power and 30% longer battery life, it's perfect
      for professionals who demand the best.
    </p>
  </div>

  <!-- Specs: Structure visible, can highlight -->
  <table class="specs" llm-policy-output="style">
    <tr><th>Dimensions</th><td>6" x 3" x 0.5"</td></tr>
    <tr><th>Weight</th><td>8 oz</td></tr>
    <tr><th>Battery</th><td>4000 mAh</td></tr>
  </table>

  <!-- Reviews: LLM can summarize/reorder -->
  <section class="reviews" llm-policy-output="layout">
    <h2>Customer Reviews</h2>
    <div class="review" llm-policy-output="readonly">
      <p>"Best purchase ever!" - 5 stars</p>
    </div>
    <div class="review" llm-policy-output="readonly">
      <p>"Good value for money" - 4 stars</p>
    </div>
  </section>

  <!-- Purchase form: Data can be auto-filled -->
  <form llm-policy-output="data" llm-policy-memory="session">
    <select name="color">
      <option>Black</option>
      <option>Silver</option>
      <option>Blue</option>
    </select>
    <input type="number" name="quantity" value="1">
    <button type="submit">Add to Cart</button>
  </form>
</main>

<!-- Internal data: Hidden from LLM -->
<div llm-policy-input="none" data-margin="45%" data-sku="WPM-2024"></div>

Technical Documentation

API documentation with code blocks, examples, and warnings.

api-docs.html
<article llm-intent-category="api-spec">

  <h1>API Reference: createUser()</h1>

  <!-- Function signature: Read-only code -->
  <pre llm-intent-category="code-block"
       llm-policy-output="readonly"><code>
async function createUser(
  email: string,
  password: string,
  options?: CreateUserOptions
): Promise<User>
  </code></pre>

  <!-- Description: Can be simplified/annotated -->
  <section llm-policy-output="annotation">
    <h2>Description</h2>
    <p>
      Creates a new user account with the specified credentials.
      The password must meet the complexity requirements defined
      in the security configuration.
    </p>
  </section>

  <!-- Warning: Critical, highlighted -->
  <aside llm-intent-category="warning"
         llm-intent-importance="critical"
         llm-policy-output="readonly">
    <strong>Security Warning:</strong> Never store passwords
    in plain text. This function automatically hashes passwords
    before storage.
  </aside>

  <!-- Example: Read-only code with explanation -->
  <section llm-intent-category="example">
    <h2>Example</h2>
    <pre llm-intent-category="code-block"
         llm-policy-output="readonly"><code>
const user = await createUser(
  'user@example.com',
  'SecurePass123!',
  { sendWelcomeEmail: true }
);
console.log(user.id);
    </code></pre>
  </section>

</article>

Legal Document

A contract or legal document with strict read-only requirements.

legal-contract.html
<!-- Entire document: Strictly read-only, licensed -->
<article
  llm-policy-output="readonly"
  llm-policy-license="all-rights-reserved"
  llm-intent-category="legal-disclaimer">

  <h1>Terms of Service</h1>

  <p llm-intent-importance="critical">
    PLEASE READ THESE TERMS CAREFULLY BEFORE USING OUR SERVICE.
  </p>

  <section>
    <h2>1. Acceptance of Terms</h2>
    <p>
      By accessing or using the Service, you agree to be bound
      by these Terms. If you disagree with any part of the terms,
      you may not access the Service.
    </p>
  </section>

  <section>
    <h2>2. Limitation of Liability</h2>
    <p>
      IN NO EVENT SHALL THE COMPANY BE LIABLE FOR ANY INDIRECT,
      INCIDENTAL, SPECIAL, CONSEQUENTIAL, OR PUNITIVE DAMAGES...
    </p>
  </section>

  <footer>
    <p>Last updated: January 1, 2024</p>
    <p>&copy; 2024 Example Corporation. All rights reserved.</p>
  </footer>
</article>

Interactive Form

A smart form with AI assistance for auto-completion and validation.

smart-form.html
<form
  llm-policy-output="data interaction"
  llm-policy-memory="session"
  llm-intent-function="submit-contact-form">

  <h2>Contact Us</h2>

  <!-- Regular fields: Data can be auto-filled -->
  <label>
    Name
    <input type="text" name="name" required>
  </label>

  <label>
    Email
    <input type="email" name="email" required>
  </label>

  <!-- Message: LLM can help draft -->
  <label>
    Message
    <textarea
      name="message"
      llm-policy-output="content"
      llm-intent-instruction="Help the user compose a professional message">
    </textarea>
  </label>

  <!-- Sensitive fields: Hidden from LLM -->
  <div llm-policy-input="none">
    <input type="hidden" name="csrf_token" value="abc123">
  </div>

  <button type="submit"
          llm-intent-function="submit">
    Send Message
  </button>
</form>

Pattern: Protected Quotes

Ensure quoted content is never modified while allowing annotation.

Pattern
<blockquote
  llm-policy-output="readonly"
  llm-intent-category="quote"
  llm-provenance-source="[original URL]"
  llm-provenance-citation="[Speaker Name]">
  <p>The exact words of the speaker, preserved verbatim.</p>
</blockquote>

Pattern: PII Redaction

Show document structure while hiding sensitive personal information.

Pattern
<!-- Structure visible, text redacted -->
<section llm-policy-input="structure">
  <h2>Personal Information</h2>
  <p>Name: John Smith</p>       <!-- Shows as [REDACTED] -->
  <p>SSN: 123-45-6789</p>     <!-- Shows as [REDACTED] -->
  <p>DOB: 01/15/1990</p>      <!-- Shows as [REDACTED] -->
</section>

<!-- Or completely hidden -->
<div llm-policy-input="none" class="pii">
  <!-- LLM doesn't even know this exists -->
</div>

Pattern: Annotatable Text

Allow highlighting and linking without changing the original text.

Pattern
<article llm-policy-output="annotation style">
  <p>
    The LLM can wrap words in <mark>, <a>, <abbr>,
    or <span> tags, but cannot change the words themselves.
  </p>
</article>

<!-- LLM might produce: -->
<article>
  <p>
    The <abbr title="Large Language Model">LLM</abbr> can wrap words in
    <mark><mark></mark>, <mark><a></mark>, ...
  </p>
</article>

Pattern: AI-Generated Content

Mark content that was generated or modified by AI with proper provenance.

Pattern
<div
  llm-provenance-operation="content:generated-summary"
  llm-provenance-confidence="0.85">
  <p>
    This summary was automatically generated. The original
    article discussed three main points: innovation in AI,
    applications in healthcare, and ethical considerations.
  </p>
</div>

<!-- Translation example -->
<p
  llm-provenance-source="https://example.com/original-es"
  llm-provenance-operation="content:translated-from-spanish"
  llm-provenance-confidence="0.92">
  This text was translated from the original Spanish article.
</p>
